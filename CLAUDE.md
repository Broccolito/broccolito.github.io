## Project Overview

This is Wanjun Gu's personal portfolio website hosted on GitHub Pages at `https://broccolito.github.io/`. It is a static site with no build step — all files are served directly.

## Deployment

Changes are deployed by pushing to the `main` branch. The `auto_sync.sh` script automates this:

```bash
bash auto_sync.sh
```

This stages all changes, commits with message "automatic commit", pulls from origin, and pushes to main.

## Architecture

**Single-page portfolio (`index.html`):** Bootstrap 5.3.3 responsive layout with a two-column design (sidebar + main content). The page features sections for About, Skills, and Software cards linking to package documentation.

**Embedded package documentation (`software/tkoi/`, `software/pkoi/`):** Pre-built static sites generated by R's `pkgdown`. These are not meant to be modified manually — they are generated from their respective R package repositories and copied here. Each contains `index.html`, `articles/`, `reference/`, and `deps/` subdirectories.

**Assets:** Profile photo at `pictures/wanjun.jpg`, CV at `Wanjun Gu CV.pdf`.

## Key Design Decisions

- No JavaScript framework or build toolchain — pure HTML/CSS with Bootstrap CDN
- Software documentation is generated externally (via `pkgdown` in R) and committed as static files
- The `software/*/deps/` directories contain vendored JS/CSS dependencies for offline-capable docs
